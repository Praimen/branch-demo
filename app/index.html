Feature Branch WorkFlow

How to Feature Branch

  • build the feature branch from the release branch
  • The only branch that should ever be pulled into your feature branch is the release branch
  • The feature branch should only contain your feature
  • The feature branch should be successfully built before integration built and testing

When to Co-Op

  • when your feature is possibly dependant on another feature, but a scenario exists in which your feature could go to PROD on its own

Release Branch WorkFlow

Monday

  • the release branch is created from master
  • all feature branches will pull the release into their branch

During the week

  • a pull request to the release branch should be created once a feature is ready for QA
  • QA Lead should be a reviewer, her approval will ensure that the feature has been tested and that there are no other commits past her approval
  • the release branch will only contain features that have been tested, have little chance for change, and slated to go to PROD on Sunday or Thursday

Before Deployment

  • Friday morning the release features going into PROD on Sunday will be merged into
  • This branch will be build and sent to QA for a final production integration testing pass, this will be the build that is send to PROD

Monday after deployment

  • the release branch is verified with the deployments that actually when to production
  • the release branch is merged into master
  • all feature branches will pull the release into their branch

Co-op Branch WorkFlow

How to Co-Op

  • build the co-op branch from the release branch
  • merge your feature into the co-op branch, then merge any other feature needed into the Co-Op
  • Co-Op testing/integration should not necessarily delay QA testing

When to Co-Op

  • When there is an issue in the Integration build but maybe not in your feature branch, co-op will allow you to troubleshoot particular features with your own
  • when your feature is possibly dependant on another feature, but a scenario exists in which your feature could go to PROD on its own
